# SQL, noSQL Injection


## TASKS

 0. SQLi - Basic Injection Discovery
    - Identify the vulnerable parameter.

 1. SQLi - Extracting Database Information
    - Find the hidden flag by extracting information from the database.

 2. SQLi - Data Exfiltration from a Specific Table
    - Find the hidden flag in one of the table of the database.

 3. SQLi - Time-Based Blind Injection
    - craft a payload that causes the server to return a flag if the query execution takes longer than 5 seconds.

 4. Second-Order Blind Injection
    - submitting a payload through a registration form that seems benign until it is processed during login, revealing a flag based on a jinja error where FLAG is an accessible variable.

 5. NoSQLi - Discovering Injection Vulnerabilities
    - detect an endpoint within the application that is susceptible to NoSQL injection but NOT Harmful

 6. NoSQLi - Bypassing Login via injection
    - find the flag by exploiting a login form.

 7. NoSQLi - Enumerating for Profit
    - identify a user profile with sufficient balance and engage in cryptocurrency exchange activities on the platform to accumulate at least 1 HBTNc coin.
